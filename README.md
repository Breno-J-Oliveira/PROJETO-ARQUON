# SPRINT 1

A Sprint 1 corresponde à fase inicial de planejamento do projeto Arqon.

Nesta etapa foram definidos os requisitos do sistema, identidade visual e estrutura inicial da plataforma.

O sistema tem como objetivo desenvolver uma plataforma digital para locação de roupas premium sustentáveis, permitindo que usuários acessem peças de marcas renomadas de forma prática e moderna.

O projeto contempla cinco páginas principais:

- Página de Login
- Cadastro de Marcas Parceiras
- Cadastro de Peças
- Painel Administrativo
- Listagem de Produtos

O objetivo é desenvolver um sistema acessível, organizado e eficiente para gerenciamento de peças de vestuário e marcas parceiras.

---

# Funcionalidades

- Login seguro com e-mail e senha
- Cadastro de marcas parceiras
- Cadastro de produtos
- Registro de informações detalhadas das peças
- Listagem organizada dos produtos cadastrados

# SPRINT 2

Na Sprint 2, a Arqon avança da prototipagem para a construção do sistema de locação de roupas premium sustentáveis, implementando o Frontend com Bootstrap com base nas telas definidas no Figma.

Ao mesmo tempo, é realizada a modelagem dos dados do sistema, criando entidades como:

- usuários
- roupas
- categorias
- tamanhos
- regras de locação
- informações de sustentabilidade

Essas estruturas formam a base para o desenvolvimento da camada Backend nas próximas sprints.

---

# Bootstrap

Bootstrap é um framework front-end gratuito e de código aberto utilizado para acelerar o desenvolvimento de sites e aplicações web.

Ele oferece componentes prontos como:

- botões
- formulários
- grids
- menus
- modais

permitindo a criação rápida de interfaces modernas e responsivas.

Além disso, o Bootstrap segue a abordagem mobile first, garantindo compatibilidade com:

- celulares
- tablets
- desktops

---

# Modelagem de Dados

A modelagem de dados do sistema foi realizada utilizando JSON e SQL.

O JSON foi utilizado para estruturar:

- usuários
- roupas
- categorias
- tamanhos
- regras de locação
- histórico de pedidos

Já o SQL foi utilizado para:

- criação de tabelas
- consultas relacionais
- armazenamento persistente
- integridade dos dados

A integração entre Frontend e Backend foi planejada para garantir escalabilidade e facilidade de manutenção do sistema.

# SPRINT 3

# Definição de PHP

PHP é uma linguagem de programação utilizada para desenvolvimento web no lado do servidor.

Com PHP é possível:

- criar sistemas dinâmicos
- processar formulários
- autenticar usuários
- conectar bancos de dados
- desenvolver APIs

A linguagem foi utilizada no projeto Arqon para construção do Backend e integração das funcionalidades do sistema.

---

# Definição de Composer

Composer é um gerenciador de dependências para PHP.

Sua função é:

- instalar bibliotecas
- atualizar pacotes
- organizar dependências
- automatizar carregamento de classes

O Composer foi utilizado para estruturar melhor o projeto e facilitar a manutenção do código.
